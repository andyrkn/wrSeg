Text Segmentation
Python, Ocropy
Goal

Implement a module that is going to be responsible for the segmentation of a page. Should return information about the data contained in the given document, such as the title, columns, rows, annotations, page number etc. Should also compute statistics about certain attributes.

Technologies used

The reasons behind our choice include the complexity of the documentation, the implementation language and readability. Out of all the other open-source libraries that were researched, Ocropy qualified as the best option at the time being. Other candidates included page_segmentation and sdalg, which were excluded to the lack of documentation. We also considered vips_java (written in Java which we considered a great plus), but the extended documentation was in Czech and we later discovered that this tool is better used on web pages, rather than handwritten pages. We also found some articles about page segmentation that looked thoroughly researched, but we couldn’t find an open-source project linked to them. (TODO: maybe add links here)
