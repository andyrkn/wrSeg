Back-End
Java, Spring
Java platform
Advantages

- we are familiar with the language, since we created applications last year within the “Advanced programming” subject;

- cross-platform

- one of the most significant advantages of Java is its ability to move easily from one computer system to another. The ability to run the same program on many different systems is crucial to World Wide Web software, and Java succeeds at this by being platform-independent at both the source and binary levels;

- object-oriented

- allows us to create modular programs and reusable code.
Disadvantages

- the Artificial Intelligence layer will be developed using Python language and thus we will not be able to run the code purely on our platform and we will execute it “remotely”, using Runtime object;

- slow-performance:

significantly slower and more memory-consuming than natively compiled languages such as C or C++;

- memory management, with Java, is a little expensive, since the Garbage Collector needs to work in the background.

Spring
The Spring Framework is an application framework and inversion of control container for the Java platform.

Some advantages, offered by this framework, are represented by:

- the “Dependency Injection” system, through which is imposed an approach which facilitates writing of tests;

- the fact in the latest versions of Spring, we can configure the application by using annotations instead of XML files

Spring can be used without XML (or very little).
If you dive in with Spring 3.0.5.GA or higher then you can use the dependency injection support from JDK6+. This means that you can wire up dependencies using the @Component and @Resourceannotations;

- Spring MVC web application and RESTful web service framework;

- It also provides a good way to structure your application into layers. Use it, it will save you a lot of time in the long run.

Spring was used in the application to create a back-end server that expands REST services.
Flow of the server-side:

At PORT 8082, you can upload a file using the following end-point: 8082/upload-file

An exception is thrown when the file doesn't have an extension.

• JPEG;

• PNG;

• SVG;

• PDF;

and other extensions that describe an image...

Flow:

⟶ a file is uploaded using the end-point indicated above;

⟶ the file is uploaded in the Back-End\assets directory (the assets directory must exist);

⟶ the AI layer goes into execution

⟶ the same end-point returns a JSON corresponding to the processing of the AI layer

Development phases

I. Ability to POST an image (the image that is uploaded on a specific end-point - /files - is being placed in a folder and from there, that file will be used by the Artificial Intelligence layer);

II. Ability to upload files of different extensions;

III. Refactoring of the code and ability to throw an exception if the image is not valid;

IV. Checking if the file extension is not a valid one and throwing an exception accordingly;

V. Ability to execute the Python script (AI layer) when the file is uploaded.

VI. The API throws exceptions according to unusual situation like ProcessedImagesReader, ProcessingException etc.

VII. Ability to log the execution of the Application. The logger logs info about the execution of the application by days (files that begin with ro-wrseg-logging from the server-side directory).

VIII. Reestablished the communication with AI Layer and refactored the code in ScriptRunner service. Ability to print the STANDARD OUTPUT of the execution of the Python Script (AI Layer).

IX. All three layers (CONTROLLER, SERVICE, REPOSITORY) of the server-side make use of the logging technology.

X. The response to request, that is being made by the Front-End, is created by using the JSON file which is created when the execution of the Python script has ended. The JSON is read and parsed accordingly and the request to the API is fulfilled.
